# System Design Interview

This folder contains full interview-style system design answers. Each document is meant to show how to clarify requirements, estimate scale, propose a defensible architecture, and explain trade-offs clearly.

## What Is Here

| Problem | File | What It Tests |
| --- | --- | --- |
| Calendar System | [01-design-calendar-system.md](01-design-calendar-system.md) | Scheduling, recurring events, consistency, time zones |
| Google Docs | [02-design-google-docs.md](02-design-google-docs.md) | Real-time collaboration, sync, conflict resolution, scaling |

## How To Use This Folder

- Use these documents to study answer structure, not just architecture diagrams.
- Pay attention to scope clarification, assumptions, and trade-offs.
- Pair this folder with `06-system-design-fundamental/` if a design references concepts you want to deepen.

## Recommended Reading Order

1. [01-design-calendar-system.md](01-design-calendar-system.md)
2. [02-design-google-docs.md](02-design-google-docs.md)
